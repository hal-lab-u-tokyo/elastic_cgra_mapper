# Direct-Grid Placement Engine

Direct-grid implementation for paper-guided YOTO/YOTT, cpu_mapping YOTO/YOTT, array PRISA, and internal SA.

## Files

- `placement2d_array_engine.cpp`: run flow, result conversion, and logging.
- `placement2d_array_engine_internal.hpp`: private state, helper structs, and method declarations.
- `placement2d_array_engine_options.cpp`: config normalization, seeds, timeouts, and mapper names.
- `placement2d_array_engine_cache.cpp`: DFG edges, grid cells, compatibility tables, I/O policy, and distance cache.
- `placement2d_array_engine_quality.cpp`: placement cost, quality metrics, ranking helpers, and validity checks.
- `placement2d_array_engine_faithful_traversal.cpp`: paper-guided YOTO/YOTT traversal order, annotations, and candidate ranking.
- `placement2d_array_engine_cpu_mapping.cpp`: cpu_mapping-style YOTO/YOTT port, type matrix, freedom grid, tips, and intersections.
- `placement2d_array_engine_prisa.cpp`: PRISA SIS, PR/WR computation, move proposals, and refinement.
- `placement2d_array_engine_sa.cpp`: internal SA placement baseline.

Each algorithm file starts with a short pipeline comment. Use that as the local
table of contents before following individual helper functions.

## Run Flow

1. `Placement2DArrayMapperBase::Execution()` forwards JSON options to `RunPlacement2DArrayEngine()`.
2. `Placement2DArrayEngine` builds caches in its constructor:
   - `BuildDFGCache()`: directed edges, predecessors, successors, and node degree.
   - `BuildGridCache()`: physical cell to MRRG node mapping, supported ops, and memory-accessible cells.
   - `BuildCompatibilityCache()`: legal cells for each DFG node.
   - `BuildPRISACache()`: PRISA resource order and distance/weak-region matrices.
3. `Placement2DArrayEngine::Run()` dispatches by `Placement2DArrayKind`:
   - paper-guided YOTO/YOTT: `RunFaithfulTraversalMultiStart()`.
   - cpu_mapping YOTO/YOTT: `RunCPUMappingMultiStart()`.
   - PRISA variants: `RunPRISAMultiSeed()`.
   - array SA: `RunSAMultiSeed()`.
4. The selected engine runs one or more trials, keeps the best placement, and converts cell IDs back to MRRG node IDs.
5. The returned mapping has empty edge routes. Routing validity is evaluated only by later analysis or modulo mappers.

## Paper-Guided YOTO/YOTT Flow

1. `BuildFaithfulTraversalPlan(use_annotations)` creates a sequence of edge-local placement steps.
2. YOTO calls it with `use_annotations=false`; YOTT calls it with `true`.
3. `ChooseFaithfulNeighbor()` controls which adjacent DFG node is visited next.
4. `PlaceFaithfulStep()` places the target near the anchor:
   - `ClosestCompatibleCells()` keeps only closest legal cells.
   - `FaithfulTraversalRank()` breaks ties by annotation priority, annotation cost, distance, degree match, and random order.
5. `ConstructFaithfulTraversalPlacement()` fills any unplaced nodes after the edge plan.
6. `RunFaithfulTraversalMultiStart()` repeats trials and keeps the lowest `PlacementCost()`.

## cpu_mapping YOTO/YOTT Flow

1. `BuildCPUMappingPlan()` ports the public `cpu_mapping` traversal order.
2. YOTO uses local adjacency placement through `PlaceCPUMappingNearNodeYOTO()`.
3. YOTT adds cycle/reconvergence annotations through `ApplyCPUMappingCycleAnnotations()` and places through `PlaceCPUMappingNearNodeYOTT()`.
4. Candidate cells are generated by:
   - `CPUMappingTipCells()` for one-hop or mesh local tips.
   - `CPUMappingCellsWithinAnnotatedDistance()` for YOTT annotation constraints.
   - `TryCPUMappingAdjacency()` as the fallback local search around the anchor.
5. `BestCPUMappingDegreeCell()` chooses among candidates using the mutable freedom grid and the optional public-code-compatible degree condition.
6. `ConstructCPUMappingPlacement()` fills remaining nodes after traversal.
7. `RunCPUMappingMultiStart()` repeats trials and keeps the lowest `PlacementCost()`.

## Edit Points

- YOTO/YOTT traversal order: `placement2d_array_engine_faithful_traversal.cpp` or `placement2d_array_engine_cpu_mapping.cpp`.
- Candidate scoring and placement cost: `placement2d_array_engine_quality.cpp`.
- I/O-node treatment and architecture compatibility: `placement2d_array_engine_cache.cpp`.
- PRISA placement behavior: `placement2d_array_engine_prisa.cpp`.
- Mapper options exposed to JSON configs: `placement2d_array_engine_options.cpp` plus `mapper/include/mapper/placement2d/placement2d_array_mapper_base.hpp`.

## Config Keys Used Here

| key | consumed by | effect |
| --- | --- | --- |
| `Algorithm.type` | mapper factory and frontend files | chooses generic search, paper-guided array, cpu_mapping, PRISA, ILP, or SA implementation |
| `max_trials` | `MaxTrials()` and multi-start loops | number of independent placement attempts per seed |
| `seed_count` | `SeedCount()` and multi-start loops | number of deterministic seed offsets |
| `random_seed` | `SeedFor()` | base seed for traversal and candidate tie-breaking |
| `max_iterations` | PRISA/SA loops | iteration budget for iterative refinement methods |
| `io_node_policy` | `IsIONode()` and `UsesStructuralIOCellTypes()` | `structural` treats source/sink-like nodes as I/O; default `opcode` uses memory opcodes |
| `cpu_mapping_bug_compatible_degree` | `BestCPUMappingDegreeCell()` | preserves or disables a public-code-compatible degree comparison quirk |
