## Comparison Scope

The papers contain two kinds of artifacts:

- Numeric tables: values can be copied into reference CSV files and compared exactly.
- Plot-only figures: exact per-benchmark values are not required for this repository workflow. Compare them through the qualitative claims and aggregate values stated in the text, then generate the corresponding local per-benchmark plots for visual inspection.

For a paper-faithful comparison, report both:

- `paper_reference`: values stated in the paper.
- `local_result`: values generated by this repository under the closest available benchmark, grid, I/O, and mapper settings.

Always record benchmark-set mismatch separately. Several current local DOTs are normalized versions and do not exactly match the YOTT paper's 23 benchmark list or node granularity.

## TRAVERSAL/YOTO Paper

### Table I

Paper artifact:

- Execution time for 15 CGRA-ME benchmarks in the modulo-scheduled ADRES setting.
- Paper location: Section V-E, Table I.

Paper setting:

- ADRES 4x4.
- `II = 2`.
- CGRA-ME command includes `--II 2`, mapper `0` for ILP, mapper `1` for SA, C++ ADRES architecture, `cols=4 rows=4`.
- Traversal reports `(All, 100)` in microseconds and stops early if a feasible solution is found.

Local support:

- The repository has modulo experiments, but current YOTO/YOTT placement2d paper comparison does not match this ADRES modulo setting.
- To compare Table I, use a separate modulo manifest with 4x4, II=2, ADRES-like I/O/memory restrictions, and include `modulo_yoto`, `modulo_yott`, ILP, and SA/heuristic rows.

Status:

- Not covered by the placement2d paper comparison.
- Needs a dedicated modulo paper-comparison manifest.
